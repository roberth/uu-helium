Helium, differences when compared to Haskell 98

  • Data type, show function
  • The module system is very simple:
    • Everything is always exported: data types, functions, synonyms, instances.
    • It is not allowed to import something via two different import declarations. This means that if you have a module A and both B and C import A and then a fourth module D imports both B and C that you will get clashes for all the names from module A.
    • You can hide functions in an import declaration with hiding; you cannot hide the show... functions that are generated for each data type. Hiding other things like types and constructors is not possible.
    • qualified imports and imports with as
  • The following character sequences are supported in characters and strings: \\, \n, \a, \b, \f, \r, \t, \v, \", \'. Other escape sequences like the ones with numbers are not supported.
  • Numeric literals are not overloaded. 3 is of type Int and 3.0 is of type Float
